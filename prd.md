## 0. 현재 MVP UX 결정

2026-05-19 기준 MVP의 첫 구현 방향은 검색 결과 목록 중심이 아니라 **자연어 입력 후 즉시 비교 화면을 보여주는 방식**으로 한다.

- 사용자는 사건을 자연어로 입력한다.
- 서비스는 입력 사건과 쟁점이 가까운 기준 판례와 결론이 다른 판례를 함께 제시한다.
- 화면은 좌측 `기준 판례`, 우측 `반대 결론 판례`의 2열 비교를 기본으로 한다.
- 유사도 퍼센트는 사용자 화면에 노출하지 않는다.
- 내부 검색 점수는 후보 선별과 정렬에만 사용한다.

이 결정은 초기 사용자 피드백을 받기 전까지의 MVP 기준이며, 피드백에 따라 검색 결과 목록형 UI를 다시 검토할 수 있다.

## 1. 제품 개요

본 서비스는 사용자가 특정 판례를 입력하면, 해당 판례와 사실관계 및 법적 쟁점이 유사한 판례를 찾아주고, 그중 결론이 다른 판례를 별도로 제시하는 판례 탐색 서비스이다.

MVP의 목적은 법률 전문가 수준의 해석을 대신하는 것이 아니라, 법을 처음 배우는 사용자가 판례를 읽은 뒤 관련 판례를 더 쉽게 확장 탐색할 수 있도록 돕는 데 있다.

### 1.1 제품이 해결하려는 문제

- 판례를 하나 읽은 뒤 어떤 판례를 추가로 찾아봐야 하는지 알기 어렵다.
- 일반적인 키워드 검색만으로는 유사한 사실관계의 판례를 찾기 어렵다.
- 유사한 사건 중 결론이 다른 판례를 찾으려면 많은 수작업이 필요하다.
- 초심자는 검색어를 어떻게 구성해야 하는지부터 막히기 쉽다.

### 1.2 MVP의 핵심 목표

1. 기준 판례와 유사한 판례를 찾는다.
2. 그중 결론이 다른 판례를 별도로 보여준다.
3. 왜 해당 판례가 후보로 제시되었는지 최소한의 근거를 보여준다.
4. 모든 결과에서 원문을 직접 확인할 수 있도록 한다.

### 1.3 MVP에서 하지 않는 것

- 법률 자문 제공
- 사건의 최종 법적 결론 제시
- 법률요건 단위의 정교한 해설
- 변론 전략 자동 생성
- 반박 논리 자동 작성
- 고급 학습 모드 및 교육용 커리큘럼

위 기능은 추후 확장 기능으로 검토한다.

## 2. 핵심 사용자

### 2.1 주요 사용자

- 법을 이제 막 접하는 사람
- 법학 입문자
- 판례 읽기와 검색에 익숙하지 않은 학습자

### 2.2 사용자 니즈

- 이 판례와 비슷한 판례를 더 보고 싶다.
- 결론이 다른 판례도 같이 보고 싶다.
- 내가 무엇을 기준으로 판례를 더 찾아봐야 하는지 감을 잡고 싶다.
- 어려운 검색식을 직접 만들지 않아도 관련 판례를 찾고 싶다.

## 3. 제품 설계 원칙

### 3.1 왜곡 최소화

- 원문 판례는 별도로 보존한다.
- LLM이 생성한 요약과 원문은 명확히 구분한다.
- 결과 화면에는 원문 확인 경로를 항상 제공한다.
- "반대 판례"라는 단정적 표현보다 "유사하지만 결론이 다른 판례"라는 표현을 기본으로 사용한다.
- LLM이 만든 설명은 판례 원문을 대체하지 않는다.

### 3.2 보수적 설명

- 초심자에게 이해를 돕되, 원문에 없는 사실을 보충하지 않는다.
- 판례 간 차이를 설명할 때는 확정적 판단보다 비교 보조의 성격을 유지한다.
- 신뢰도가 낮은 처리 결과는 별도 상태로 관리한다.

### 3.3 MVP 우선순위

- 풍부한 기능보다 검색 정확도와 신뢰도를 우선한다.
- 초기에는 사용자 경험을 넓히기보다 핵심 검색 플로우를 완성한다.

## 4. LLM의 역할 정의

### 4.1 MVP에서 LLM이 수행하는 일

1. 판례 원문에서 검색에 필요한 최소 정보 추출
   - 핵심 쟁점
   - 사실관계 요약
   - 판결 결과 라벨
2. 판례 간 간단한 비교 설명 생성
   - 어떤 점이 비슷한지
   - 결론이 어떻게 다른지
3. 긴 텍스트를 검색 가능한 구조로 정리

### 4.2 MVP에서 LLM이 수행하지 않는 일

- 새로운 법적 판단 생성
- 원문에 없는 사실 추가
- 특정 판례가 "정답"이라고 단정
- 법률 자문 또는 소송 전략 제시
- 사용자를 대신한 최종 결론 도출

### 4.3 LLM 사용 원칙

- 가능하면 수집 및 전처리 단계에서 1회 처리하고, 검색 시에는 저장된 정제 데이터를 재사용한다.
- 프롬프트 버전과 처리 결과를 저장하여 재현 가능성을 확보한다.
- 판결 결과 라벨은 품질 검수 대상으로 별도 관리한다.

#### B. 검색용 구조화 데이터

- 핵심 쟁점 요약
- 사실관계 요약
- 판결 결과 라벨
- 사건 유형 또는 법영역

#### C. 검색용 벡터 데이터

- 사실관계 임베딩

### 5.3 저장 원칙

- 추후 품질 개선을 위해 LLM 처리 버전과 검수 상태를 함께 저장한다.

## 6. 판례 검색 방식

### 6.1 검색 설계 원칙

MVP 검색은 실제 법률가의 판례 탐색 습관을 단순화해 반영한다.

변호사들은 일반적으로 다음 순서를 중요하게 본다.

1. 어떤 법적 쟁점의 사건인지
2. 어떤 조문이 적용되는지
3. 사실관계가 얼마나 유사한지
4. 결론이 같은지 다른지

MVP는 이 흐름을 검색 로직에 반영한다.

### 6.2 사용자 입력 방식

#### 1. 사건번호 입력

- 사용자가 사건번호를 입력하면 해당 판례를 기준 판례로 사용한다.
- 초기 MVP에서 가장 우선 지원한다.

#### 2. 자연어 설명 입력

- 사용자가 사건을 간단히 설명하면, 자연어를 임베딩하여 유사 판례를 찾는다.
- 사건번호를 모르는 초심자를 위한 보조 입력 방식으로 제공한다.

#### 후순위 검토

- 판례 전문 붙여넣기는 비용과 처리 복잡도가 높아 MVP 후순위로 둔다.

### 6.3 MVP 검색 로직

1. 같은 법영역 또는 사건유형의 판례를 우선 후보로 삼는다.
2. 같은 참조조문을 가진 판례를 우선적으로 찾는다.
3. 핵심 쟁점과 사실관계 요약을 기반으로 벡터 유사도 검색을 수행한다.
4. 검색 결과를 판결 결과 라벨 기준으로 나눈다.
   - 유사한 판례
   - 유사하지만 결론이 다른 판례
5. 최종 결과를 유사도 기준으로 정렬한다.

### 6.4 유사성 판단 기준

MVP에서는 아래 3가지 축을 사용한다.

1. 참조조문 일치 여부
2. 핵심 쟁점 유사도
3. 사실관계 유사도

초기 가중치는 실험값으로 두고, 운영 중 사용자 피드백을 통해 조정한다.

### 6.5 결과 분류

- 유사 판례
- 유사하지만 결론이 다른 판례
- 검색 제외 판례

## 7. 사용자 경험

### 7.1 기본 플로우

1. 사용자가 사건번호 또는 자연어 설명을 입력한다.
2. 시스템이 기준 판례 또는 검색 의도를 파악한다.
3. 유사 판례 목록을 보여준다.
4. 결론이 다른 판례를 별도 섹션으로 보여준다.
5. 사용자는 원문 링크를 통해 직접 판례를 확인한다.

### 7.2 주요 화면

#### 1. 검색 화면

- 사건번호 입력
- 자연어 검색 입력
- 간단한 안내 문구

#### 2. 결과 목록 화면

- 사건번호
- 선고일
- 판결 결과
- 유사도 점수
- 한 줄 쟁점 요약
- 원문 링크

#### 3. 비교 화면

- 기준 판례와 선택 판례를 나란히 표시
- 공통점 간단 요약
- 다른 결론 여부 표시
- 짧은 비교 설명
- 원문 링크

### 7.3 엣지 케이스

- 결과가 없을 때: 검색 범위를 넓혀보라는 안내 제공
- 사건번호를 찾지 못할 때: 입력 확인 안내 및 원문 검색 링크 제공
- 유사도는 높지만 결론이 다른 경우: 원문 확인 권고 표시
- LLM 설명 생성 실패 시: 검색 결과는 유지하고 설명만 재시도 가능하게 처리

## 8. 품질 관리

### 8.1 핵심 품질 기준

- 판결 결과 라벨 정확도
- 유사 판례 검색 품질
- 결론이 다른 판례 분류 정확도

### 8.2 검수 정책

- 배치 처리 후 무작위 샘플을 사람이 검토한다.
- 판결 결과 라벨 오류율이 일정 기준을 넘으면 해당 배치를 재처리한다.
- 특정 라벨이 과도하게 몰리면 프롬프트 또는 분류 로직을 점검한다.

### 8.3 사용자 피드백

MVP에서는 아래 정도의 간단한 피드백만 수집한다.

- 이 판례가 실제로 유사하다고 느껴졌는가
- 결론 분류가 맞는가
- 결과가 도움이 되었는가

오류 신고가 누적된 판례는 재검수 대상으로 등록한다.

## 9. 법적 고지 및 책임 제한

- 본 서비스는 학습 보조 목적이며, 법률 자문을 제공하지 않는다.
- LLM이 생성한 요약 및 비교 설명은 참고 정보이며 원문 판례를 대체하지 않는다.
- 모든 결과 화면에 원문 확인 권고를 표시한다.
- 실제 법률 판단이 필요한 경우 전문가 검토가 필요함을 명시한다.

예시 고지:

> 본 서비스의 요약 및 비교 설명은 학습 보조 목적의 참고 정보입니다. 실제 법률 판단을 대체하지 않으며, 반드시 원문 판례를 직접 확인해야 합니다.
