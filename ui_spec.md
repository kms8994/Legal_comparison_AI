## 1. UI/UX 목표

본 서비스의 UI/UX는 법을 처음 접하는 사용자가 부담 없이 검색을 시작하고, 결과를 빠르게 이해하며, 필요한 경우 원문 판례까지 자연스럽게 이동할 수 있도록 돕는 것을 목표로 한다.

현재 MVP 화면은 검색 결과 목록형이 아니라 **자연어 입력 기반 비교 워크스페이스**를 기준으로 한다. 사용자가 사건을 자연어로 입력하면 좌측에는 `기준 판례`, 우측에는 `반대 결론 판례`를 바로 보여주고, 아래에 `핵심 차이`, `공통점`, `짧은 비교 설명`을 제공한다.

### 1.1 핵심 목표

1. 처음 방문한 사용자가 3초 안에 무엇을 할 수 있는 서비스인지 이해한다.
2. 유사 판례와 유사하지만 결론이 다른 판례의 차이가 즉시 보인다.
3. 결과 화면은 많은 정보를 담되 복잡하게 느껴지지 않는다.
4. 서비스 전체가 신뢰감 있고 세련되게 보인다.
5. 모바일에서도 검색과 비교가 불편하지 않다.

## 2. 디자인 방향

### 2.1 제품 톤

> 조용하고 세련된, 학습 친화적인 법률 검색 도구

### 2.2 지향하는 인상

- 신뢰감 있는
- 명확한
- 정돈된
- 지적인
- 초심자에게 부담스럽지 않은

### 2.3 피해야 할 인상

- 지나치게 무겁고 권위적인 법률사무소 스타일
- 과하게 화려한 그라데이션 중심 UI
- 카드가 과도하게 많은 SaaS 대시보드 스타일
- 너무 가볍고 장난스러운 교육 서비스 스타일

## 3. UX 원칙

### 3.1 정보 우선

- 장식보다 정보 구조를 우선한다.
- 사용자가 다음 행동을 고민하지 않도록 화면 흐름을 단순하게 만든다.
- 각 화면에서 가장 중요한 정보가 먼저 보이도록 배치한다.

### 3.2 초심자 친화

- 검색 방식이 어렵게 느껴지지 않아야 한다.
- 필요한 설명은 짧게 제공하되, 화면을 설명문으로 채우지 않는다.
- 법률 정보는 단순화하되 왜곡하지 않는다.

### 3.3 신뢰성

- 원문 링크를 항상 제공한다.
- 판결 결과와 유사도는 명확하게 표시한다.
- LLM 설명은 보조 정보임을 자연스럽게 드러낸다.

### 3.4 일관성

- 같은 의미의 정보는 같은 위치와 같은 스타일로 반복한다.
- 색상, 배지, 버튼, 간격, 타이포그래피 규칙을 화면 전체에서 일관되게 사용한다.

## 4. 시각 시스템

### 4.1 컬러 팔레트

#### 기본 색상

| 용도 | 색상 제안 |
| :--- | :--- |
| 배경 | `#F7F8FA` |
| 표면 | `#FFFFFF` |
| 기본 텍스트 | `#1F2937` |
| 보조 텍스트 | `#6B7280` |
| 경계선 | `#E5E7EB` |

#### 브랜드 및 상태 색상

| 용도 | 색상 제안 |
| :--- | :--- |
| Primary | `#1D4ED8` |
| Primary Hover | `#1E40AF` |
| 유사 판례 | `#0F766E` |
| 결론이 다른 판례 | `#B45309` |
| 경고 | `#B91C1C` |
| 성공 | `#15803D` |

### 4.2 색상 사용 원칙

- 한 화면을 특정 색상 하나로 과도하게 지배하지 않는다.
- `유사 판례`와 `결론이 다른 판례`는 색만으로 구분하지 않고 텍스트 라벨도 함께 사용한다.
- 주요 버튼 외에는 강한 색을 제한적으로 사용한다.
- 법률 서비스 특성상 채도가 지나치게 높은 색은 피한다.

### 4.3 타이포그래피

#### 기본 방향

- 한글과 숫자 가독성이 좋은 산세리프 계열 사용
- 제목은 간결하고 또렷하게
- 본문은 충분한 줄간격과 안정적인 크기 유지

#### 권장 체계

| 용도 | 크기 |
| :--- | :--- |
| Page Title | 28px |
| Section Title | 20px |
| Card Title | 16px |
| Body | 15px |
| Caption | 13px |

### 4.4 간격 체계

- 기본 4px 단위 스케일 사용
- 권장 간격: `4 / 8 / 12 / 16 / 24 / 32 / 48`
- 섹션 간 간격은 충분히 두고, 카드 내부 간격은 과하지 않게 유지한다.

### 4.5 모서리 및 그림자

| 요소 | 권장값 |
| :--- | :--- |
| 카드 radius | 8px |
| 입력창 radius | 8px |
| 버튼 radius | 8px |
| 그림자 | 매우 약하게 사용, 또는 경계선 중심 |

## 5. 핵심 컴포넌트

### 5.1 검색 입력

#### 구성

- 입력 모드 탭
  - 사건번호
  - 자연어 설명
- 검색 입력창
- 검색 버튼
- 예시 문구

#### UX 규칙

- 검색창은 화면에서 가장 먼저 눈에 들어와야 한다.
- 사용자가 입력 형식을 몰라도 예시만 보고 시작할 수 있어야 한다.
- 사건번호 입력과 자연어 입력은 하나의 화면 안에서 쉽게 전환 가능해야 한다.

### 5.2 결과 카드

#### 필수 정보

- 사건번호
- 선고일
- 판결 결과
- 유사도 점수
- 한 줄 쟁점 요약
- 참조조문 일부
- 원문 보기
- 비교하기

#### UX 규칙

- 사용자가 한눈에 훑을 수 있도록 정보 순서를 고정한다.
- 판결 결과와 유사도는 항상 같은 위치에 둔다.
- 비교 버튼은 명확한 액션으로 보이게 한다.

### 5.3 결과 섹션

#### 섹션 종류

- 유사 판례
- 유사하지만 결론이 다른 판례

#### UX 규칙

- 두 섹션은 시각적으로 명확히 구분한다.
- `결론이 다른 판례`는 더 강한 대비를 줄 수 있지만, 위협적으로 보이지 않아야 한다.
- 사용자가 첫눈에 두 결과군의 차이를 이해할 수 있어야 한다.

### 5.4 배지

#### 사용 예시

- `유죄`
- `무죄`
- `인용`
- `기각`
- `유사 판례`
- `다른 결론`

#### UX 규칙

- 색상과 텍스트를 함께 사용한다.
- 배지는 정보를 보조해야 하며, 본문보다 눈에 띄어서는 안 된다.

### 5.5 경고 및 고지 배너

#### 사용 예시

- 원문 확인 권고
- 유사도는 높지만 결론이 다른 결과
- LLM 설명 생성 실패

#### UX 규칙

- 고지는 상단 또는 관련 정보 바로 근처에 둔다.
- 지나치게 강한 경고 스타일은 남용하지 않는다.

## 6. 화면 설계

## 6.1 검색 화면

### 목표

- 사용자가 곧바로 검색을 시작할 수 있게 한다.

### 구조

```text
[상단 로고/서비스명]

[짧은 서비스 설명]

[입력 모드 탭]
[검색 입력창                  ][검색 버튼]
[예시 입력 문구]

[짧은 보조 설명 또는 신뢰 요소]
```

### 핵심 UX

- 첫 화면에서 설명보다 입력 행동이 중심이어야 한다.
- 입력 모드 전환은 탭으로 단순하게 제공한다.
- 지나치게 많은 기능을 노출하지 않는다.

---

## 6.2 결과 목록 화면

### 목표

- 검색 결과를 빠르게 훑고, 비교할 판례를 선택하게 한다.

### 구조

```text
[기준 판례 요약]
[정렬 / 필터 옵션]

[유사 판례 섹션]
  [결과 카드]
  [결과 카드]

[유사하지만 결론이 다른 판례 섹션]
  [결과 카드]
  [결과 카드]
```

### 기준 판례 요약 영역

- 사건번호
- 선고일
- 판결 결과
- 한 줄 쟁점 요약

### 결과 카드 우선순위

1. 사건번호
2. 판결 결과
3. 유사도
4. 쟁점 요약
5. 참조조문
6. 액션 버튼

### 핵심 UX

- 두 결과군은 섹션 수준에서 구분한다.
- 사용자가 각 결과를 클릭하기 전에도 어느 정도 비교 가능해야 한다.
- 결과 수가 많아져도 스캔성이 떨어지지 않게 한다.

---

## 6.3 비교 화면

### 목표

- 두 판례가 왜 함께 제시되었는지 빠르게 이해하게 한다.

### 데스크톱 구조

```text
[기준 판례 요약]    [비교 판례 요약]

[공통점]
[차이점]
[짧은 비교 설명]

[원문 보기 버튼들]
```

### 모바일 구조

```text
[판례 A / 판례 B 탭]

[선택한 판례 요약]

[공통점]
[차이점]
[짧은 비교 설명]
```

### 핵심 UX

- 데스크톱에서는 나란히 비교가 가능해야 한다.
- 모바일에서는 억지로 2열을 유지하지 않고 탭 전환으로 바꾼다.
- 설명은 짧고 구조화되어야 한다.

---

## 7. 반응형 설계

### 7.1 데스크톱

- 넓은 화면을 활용해 기준 판례와 비교 판례를 나란히 제시한다.
- 결과 목록은 충분한 폭을 가지되, 줄 길이는 과도하게 길어지지 않게 제한한다.

### 7.2 태블릿

- 검색과 결과 목록은 1열 중심으로 유지한다.
- 비교 화면은 필요 시 2열을 유지하되, 좁아지면 자동 전환한다.

### 7.3 모바일

- 검색창, 버튼, 필터는 세로 스택
- 결과 카드는 한 열
- 비교 화면은 탭 전환 방식
- 중요한 정보가 접히지 않고 먼저 보이도록 재배치

## 8. 빈 상태 및 오류 상태

### 8.1 검색 결과 없음

- 메시지: `검색 결과가 없습니다. 검색 범위를 조금 넓혀볼까요?`
- 보조 액션:
  - 다시 검색
  - 입력 수정

### 8.2 사건번호를 찾지 못함

- 메시지: `사건번호를 찾지 못했습니다. 입력을 다시 확인해 주세요.`
- 보조 액션:
  - 다시 입력
  - 원문 검색 링크

### 8.3 비교 설명 생성 실패

- 메시지: `비교 설명을 불러오지 못했습니다. 검색 결과는 그대로 확인할 수 있습니다.`
- 보조 액션:
  - 다시 시도

### 8.4 로딩 상태

- 단순 스피너보다 스켈레톤 UI를 우선 사용한다.
- 결과 카드의 구조를 유지한 채 로딩되도록 한다.

## 9. 접근성 원칙

- 색상만으로 상태를 구분하지 않는다.
- 키보드만으로 주요 기능을 사용할 수 있어야 한다.
- 버튼, 탭, 입력창에는 명확한 포커스 상태가 있어야 한다.
- 본문 대비를 충분히 확보한다.
- 모바일에서 터치 영역은 충분히 넓게 유지한다.

---

## 10. 인터랙션 원칙

- 사용자가 클릭 가능한 요소는 분명하게 보여야 한다.
- hover 효과는 절제하되, 반응은 분명해야 한다.
- 비교하기, 원문 보기 등 주요 액션은 한 번에 이해되어야 한다.
- 애니메이션은 정보 전달을 돕는 범위에서만 사용한다.

## 14. 최종 UI 방향

> 본 서비스의 UI는 법률 서비스다운 신뢰감과 학습 도구다운 접근성을 함께 갖춘, 조용하고 세련된 검색 경험을 지향한다.  
> 사용자는 설명을 읽기 전에 먼저 검색을 시작할 수 있어야 하고, 결과를 보는 순간 어떤 판례가 유사하며 어떤 판례가 결론이 다른지 빠르게 이해할 수 있어야 한다.
